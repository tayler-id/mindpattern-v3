Agentic Change Warranty

Turning every AI-authored pull request into an insurable event — the white space between AI code verification and AI insurance, and the technology to build into it.

VERDICT

Genuine, defensible white space. No company underwrites, warrants, or risk-scores AI-authored code changes. Incumbents insure model outputs (hallucinations, accuracy, drift, agent decisions) — never the code artifact. Verification tools provide signal with zero risk transfer. The proven, viable path is an MGA/warranty layer backed by a reinsurer, priced on an evidence/certification layer rather than actuarial loss data.

The structural gap

Two crowded camps, an empty middle

The market has a verification camp (signal, no money on the line) and an insurance camp (money, but only on model behavior). The product is the bridge: claim-grade evidence + risk transfer on the code artifact itself.

Verification camp

Signal, no risk transfer

  • SonarQube AI Code Assurance — labeling, quality gate, badge. Zero warranty language.
  • Qodo — review & governance. No risk transfer.
  • CodeRabbit — PR/IDE/CLI review. "Do the final 10%."
  • Graphite · Greptile · Diamond

Insurance camp

Money, but only on model behavior

  • Armilla AI — pays when a model fails contractual KPIs (accuracy/bias).
  • Munich Re aiSure — parametric: model error rate > threshold.
  • Corgi — E&O for the AI vendor, not the buyer's repo.
  • Testudo (2026) · Greenlight Re · Swiss Re (capacity)

Why now

Demand pressure is structural — and insurers are retreating

AI code is persistently defective

Veracode 2025: 45% of AI-generated code introduced OWASP Top-10 vulns. A Spring 2026 follow-up found newer/larger models (GPT-5.x, Gemini 3, Claude 4.5/4.6) didn't improve — flat ~55% over two years. The risk won't self-resolve. [3-0 / 2-1]

Insurers are carving AI out

ISO added CG 40 47 / 40 48 / 35 08 generative-AI exclusions (Jan 2026); Berkley added "absolute" AI exclusions to D&O/E&O. Cyber insurers exclude or reprice AI — opening the gap, not closing it. [3-0]

The blueprint is proven

Armilla = first Lloyd's MGA/Coverholder for AI liability. Doesn't carry risk (Swiss Re, Greenlight Re, Chaucer back it). Prices via "500+ evaluations", not claims data. $25M limits by Jan 2026. [3-0]

Vendor indemnities prove appetite

Copilot / Microsoft / Google / Amazon already indemnify IP claims — validating demand, but covering only copyright. The security / incident / license risk of the change is unfilled. [context]

The opportunities

Six products that fall out of the gap

Ordered lowest-risk / buildable-now → highest-risk / highest-ceiling. Ship #1 → #2 → #4 first; they generate the loss data the warranty (#3) needs.

1

The Underwriting Evidence Layer — the wedge

GitHub-native. For every AI-authored PR, emit a claim-grade evidence package: provenance attestation + diff-risk score + test evidence + model/tool trace + blast-radius — signed, tamper-evident, machine-readable. No financial promise yet → no licensing. Pure risk intelligence that bridges verification and insurance. Moat: "audit trail ≠ evidence."

Build: Low–MedMarket: MedMoat: Med–HighMVP: weeks
2

Underwriter-Pricing API

Sell the evidence + risk score to the insurers (Armilla, Munich Re, Corgi, Testudo, cyber carriers). They're retreating because they can't measure AI-code risk — you sell the measurement. The "arms dealer" position: no underwriting risk, upstream of whoever wins.

Build: LowMarket: Med–HighMoat: MedMVP: weeks–mo
3

The Agentic Change Warranty

The headline product. Attach a financial warranty to qualifying PRs: tier → eligibility → certificate. "Insurable up to $250k if merged after these two checks" — coverage conditional on risk-reducing actions. Warranty-first, MGA + reinsurer. Start narrow; earn the $1M tier with data.

Build: HighMarket: HighMoat: HighMVP: quarters
4

AI Bill of Materials for commits

Graded provenance for AI changes: who/which model, which tools, which agent, what review — signed with sigstore, tiered to SLSA Build Levels, exported as an attestation. The primitives are free but fragmented; nobody packages an "AIBOM for code changes." Feeds #1–#3; AI cyber riders increasingly demand it.

Build: Low–MedMarket: MedMoat: MedMVP: weeks
5

Coverage-Gap / Compliance product

For companies suddenly excluded by ISO CG 40 47/48 or Berkley "absolute" exclusions. Help them document their AI-code posture to negotiate coverage back, or buy a narrow warranty for the carved-out exposure. Sell through brokers who now have a gap to explain.

Build: LowMarket: MedMoat: Low–MedMVP: weeks
6

Warranty-backed channel for AI vendors

White-label the warranty so coding-agent platforms, consultancies, and dev shops can sell "our agentic work is warrantied" into conservative enterprises — the procurement/legal/cyber buyers who trust indemnity over dashboards. Fills the non-IP half vendor indemnities leave open.

Build: Med–HighMarket: MedMoat: Med–HighMVP: quarters

Go-to-market sequencing

Instrument first, insure later

Now

#1 Evidence layer + #4 AIBOM

Ship the GitHub App with no financial promise. Get adoption. Start gathering the merge→incident outcome data the actuarial problem demands. Low build & market risk.

Next

#2 Pricing API (arms dealer)

Monetize the scoring by selling it to carriers — without taking underwriting risk. You become the data/risk rail beneath whoever wins the insurance layer.

Later

#3 Warranty → #5 / #6

Take underwriting risk only once calibrated. Narrow, high-attribution PR classes first. MGA + reinsurer for the catastrophe tail — mandatory, not optional.

The hard parts

Feasibility risks & open questions

Attribution / causation — hardest problem

Proving a specific AI-authored diff caused a later breach (vs. model insurers' clean parametric "error rate > X"). Build this prototype first.

Actuarial data scarcity

Limited loss data + asymmetric info make AI risk "genuinely difficult to underwrite" (Geneva Assoc., 600 respondents). Mitigate with evidence-based pricing, like Armilla. [3-0]

Correlated / systemic risk

One bad model release or poisoned dependency can break thousands of repos at once. A catastrophe-correlated tail → reinsurance is mandatory.

Adverse selection

The riskiest PRs seek coverage. Counter it by making coverage conditional on required checks.

Buyer-pull demand is inferred, not proven

No source evidenced enterprises actually paying for per-PR code liability transfer. #1 validation target.

Stealth entrants not ruled out

"No competitor found" ≠ "none exists." Lloyd's Lab, Vouch, Coalition, CFC, Testudo not exhaustively cleared. Re-run research in 60–90 days.

Bottom line. The idea is possible and the gap is real. The engineering is tractable with off-the-shelf primitives. The risk concentrates in actuarial science and causation, not code. So: build the evidence/scoring layer now (no licensing, real value, generates the data you lack), monetize via a pricing API, and gate the financial warranty behind narrow scope + a reinsurer partner + accumulated loss data — exactly the path Armilla walked.
